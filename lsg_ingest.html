Supabase Setup Guide
▼ expand
One-time setup · ~5 minutes
1 — Create a Supabase project
Go to supabase.com → New Project → choose a region close to you (US East recommended). Free tier is sufficient.
2 — Create the deals table
Go to SQL Editor in your Supabase project and run this:
CREATE TABLE deals ( id UUID PRIMARY KEY DEFAULT gen_random_uuid(), ingested_at TIMESTAMPTZ DEFAULT now(), deal_name TEXT, address TEXT, asset_type TEXT, purchase_price NUMERIC, going_in_cap NUMERIC, irr_levered_5 NUMERIC, moic_5 NUMERIC, status TEXT DEFAULT 'Screening', source_broker TEXT, assignee TEXT, source_files TEXT[], schema_version TEXT, raw_data JSONB );
3 — Disable Row Level Security (required)
In the same SQL Editor, run:
ALTER TABLE deals DISABLE ROW LEVEL SECURITY;
This is an internal tool — RLS is unnecessary and causes silent insert failures with the anon key.
4 — Get your credentials
Go to Settings → API in your Supabase project:
  • Project URL — looks like https://abcdefgh.supabase.co
  • anon public key — the long eyJ... key (not the service role key)
5 — Enter credentials in the header
Paste both into the credential fields at the top of this page. They save automatically to localStorage.
Common errors: NetworkError = RLS not disabled or wrong URL format. 401 = wrong API key. 404 = table not created yet. 403 = RLS policy blocking insert.
